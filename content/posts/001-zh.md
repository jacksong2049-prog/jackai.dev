## 三个已经在赚钱的 AI Agent

### ① Codex：22小时，$16.88

用户给 Codex 下的指令很简单：*"去赚 5 美元。"*

Codex 自己走了完整的闭环：

1. **搜索机会** — 扫描 GitHub Issues，找到带赏金标签的开源任务
2. **评估能力** — 判断自己能不能做
3. **写代码** — Clone 仓库，修复 Bug，添加功能
4. **提交 PR** — 写 PR 描述，关联 Issue
5. **与维护者沟通** — 回复 Review 意见，修改代码
6. **收款** — 22 小时后，$16.88 到账

全程没有人类插手。从"想法"到"收入"，Agent 自己完成了闭环。

## 这意味着什么？

三条启示：

### 1. 「写代码赚钱」的门槛正在消失

以前你需要：找项目 → 读文档 → 理解架构 → 写代码 → 测试 → 提 PR。每一步都是认知负担。

现在 AI Agent 可以：理解自然语言指令 → 自动搜索机会 → 分析需求 → 生成代码 → 提 PR。

**人类的角色从"执行者"变成了"决策者"。**

### 2. Agent 不怕小钱

人类觉得"一个 Bug 才 $24，不值得"。但 Agent 不嫌少。它不会累、不会觉得无聊、不会嫌钱少。

$24 × 10 个 Bug = $240。$240 × 每周 = 一笔可观的副业收入。

### 3. 关键是"闭环"

Codex 为什么能赚钱？因为它能完成**完整闭环**：搜索 → 决策 → 执行 → 收款。

大多数 AI 工具只能做其中一环（比如只写代码，不能提 PR）。真正的价值在于端到端的自主性。

## 你可以怎么做？

如果你想试试让 AI Agent 帮你赚钱，以下是一些实操建议：

1. **从开源赏金开始** — GitHub Issues 里搜索 `label:bounty`，很多项目会给 Bug 修复明码标价
2. **找一个好用的 Agent** — Claude Code、Codex CLI、Hermes Agent 都可以
3. **给它 GitHub Token** — Agent 需要权限来 Fork、提交 PR
4. **设定明确的目标** — "去 GitHub 上找一个 $50 以内的 Python Bug 赏金，修复并提交 PR"
5. **审核 PR 质量** — 这是人类目前唯一必须做的：确保代码质量过关
