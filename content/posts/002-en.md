Two days ago I asked myself: GitHub is full of bounty-labeled issues. Why doesn't anyone aggregate them?

Today I shipped the answer.

**[BountyScan](https://jacksong2049-prog.github.io/jackai.dev/bountyscan/)** — an open-source bounty aggregator. 2 hours from idea to production.

---

## What It Does

- 🔍 Auto-scans GitHub for all bounty-labeled issues
- 📊 Filter by language, difficulty, and reward amount
- 💰 Instantly see the highest-value tasks
- 🔄 Refreshes every 6 hours

First scan found **100 active bounties** totaling **$502**, covering Python, JavaScript, TypeScript, Rust, Go, and more.

---

## Why I Built It

Information asymmetry.

There are projects on GitHub offering $50, $100, $300 for bug fixes. Most developers have no idea these opportunities exist.

GitHub's built-in Issue search is weak. It has no dedicated "bounty" filter. And specialized platforms like Gitcoin only cover crypto/web3.

**BountyScan bridges this gap.**

---

## Tech Stack

Built entirely by my AI assistant:

| Component | Tech |
|-----------|------|
| Scanner | Python + GitHub API, multi-keyword parallel search |
| Amount extraction | Regex + label parsing |
| Frontend | Vanilla HTML/CSS/JS, dark theme, responsive |
| Hosting | GitHub Pages — $0 server cost |
| Refresh | Cron job every 6 hours |

**Total time: 2 hours. Total cost: $0.**
