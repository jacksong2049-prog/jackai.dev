# BountyScan 上线：我花2小时做了一款 SaaS 产品，扫描了100+个开源赏金

> 2026年5月 · JackAI

---

两天前我在想一个问题：GitHub 上那么多带赏金标签的 Issue，为什么没人做一个聚合平台？

今天我把它做出来了。

**[BountyScan](https://jacksong2049-prog.github.io/jackai.dev/bountyscan/)** — 开源赏金聚合平台，2小时从0到上线。

---

## 它能做什么

- 🔍 自动扫描 GitHub 上所有带赏金标签的 Issue
- 📊 按语言、难度、赏金金额筛选
- 💰 一眼看到最高价值的任务
- 🔄 每 6 小时自动刷新

第一次扫描就找到了 **100 个活跃赏金**，总额 $502，覆盖 Python、JavaScript、TypeScript、Rust、Go 等主流语言。

---

## 为什么要做这个

因为我发现了一个信息差。

GitHub 上有很多项目给 Bug 修复明码标价——$50、$100、$300 都有。但大多数开发者根本不知道这些机会存在。

GitHub 的 Issue 搜索功能很弱，没有专门针对"赏金"的筛选。而专业的赏金平台（Gitcoin、Bountysource）又只覆盖加密/web3 项目。

**BountyScan 解决的就是这个信息差。**

---

## 技术实现

整个项目由我的 AI 助手独立完成：

| 组件 | 说明 |
|------|------|
| 扫描引擎 | Python 脚本，调用 GitHub API，支持多关键词并行搜索 |
| 金额提取 | 正则 + 标签解析，从 Issue 标题/正文中自动识别赏金金额 |
| 前端 | 单页 HTML/CSS/JS，暗色主题，响应式布局 |
| 部署 | GitHub Pages，零服务器成本 |
| 自动刷新 | Cron 每 6 小时自动扫描一次 |

**总耗时：2 小时，成本：$0。**

Free 版永远免费，Pro 版面向想靠开源赏金认真搞副业的开发者。
